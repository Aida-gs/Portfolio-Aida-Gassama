# Projet 3 – Requêtez une base de données avec SQL

## Description du projet  
Dans ce projet, j’ai travaillé sur l’interrogation d’une base de données relationnelle à l’aide du langage SQL, dans le cadre de ma formation Data Analyst chez OpenClassrooms. L’objectif était de construire des requêtes pertinentes (jointures, filtres, agrégations, etc.) pour extraire des insights exploitables pour l’analyse de données.


## Objectifs  
- Comprendre la modélisation relationnelle (tables, clés primaires/étrangères)
- Construire des requêtes SQL de base avec `SELECT`, `WHERE`
- Utiliser des jointures pour relier plusieurs tables (INNER JOIN, LEFT JOIN, etc.)
- Appliquer des conditions de filtrage avancées (par exemple avec `LIKE`)
- Agréger et regrouper les données selon des critères métiers (GROUP BY, fonctions d’agrégation).  
- Optimiser les requêtes pour extraire efficacement des données pertinentes.

## Compétences développées  
- **Modélisation des données** : compréhension des relations entre les tables, des clés étrangères.  
- **Langage SQL** : maîtrise de `SELECT`, `WHERE`, `JOIN`, `GROUP BY`, `HAVING`, `ORDER BY`, etc.  
- **Analyse de données** : extraire des sous-ensembles utiles, identifier des tendances, des comportements.  

## Outils utilisés  
- **Système de gestion de base de données relationnelle (SGBDR)**
- **SQL** 
- **Environnement de requêtage (client SQL)** : MySQL

## Résultats  
- Plusieurs requêtes SQL réalisées pour répondre à des cas d’usage concrets (filtrer des enregistrements, faire des jointures, agréger des données, etc.).  
- Extraction de données précises issues du croisement de tables : par exemple, des rapports sur des ventes, des utilisateurs, des segments clients, selon le schéma de la base.  
- Documentation des requêtes : je note les hypothèses métier, la logique des jointures, et les résultats attendus vs observés.


## Conclusion  
Ce projet a été essentiel pour renforcer mes compétences en **SQL** et en modélisation de bases de données. Il m’a permis de développer une méthodologie rigoureuse pour interroger efficacement des bases relationnelles, ce qui est fondamental dans le travail d’un Data Analyst. Grâce à cela, je suis capable de formuler des requêtes précises, d’extraire des données pertinentes et de poser des bases solides pour des analyses de données plus avancées.
