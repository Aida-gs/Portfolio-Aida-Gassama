# Projet 5 – Créez et utilisez une base de données immobilière avec SQL

## Description du projet  
L'objectif de ce projet était de concevoir et d’utiliser une base de données relationnelle dédiée à l’immobilier. J’ai appris à structurer les données, créer les tables, définir les relations, et interroger la base pour obtenir des informations pertinentes pour la gestion et l’analyse d’un parc immobilier.

## Objectifs  
- Concevoir une base de données relationnelle adaptée au domaine immobilier.  
- Créer les tables avec les bonnes relations (clés primaires et étrangères).  
- Insérer et gérer des données représentant des biens, des clients et des transactions.  
- Réaliser des requêtes SQL pour répondre à des questions métiers (ventes, locations, clients, zones géographiques).  
- Extraire des insights exploitables pour la prise de décision.

## Compétences développées  
- **Modélisation relationnelle** : création de tables, définition des clés primaires et étrangères, normalisation.  
- **Langage SQL** : création de tables (`CREATE TABLE`), insertion de données (`INSERT`), requêtes de sélection (`SELECT`), filtres (`WHERE`), jointures (`JOIN`), agrégations (`GROUP BY`).  
- **Gestion et manipulation de données** : mise à jour (`UPDATE`), suppression (`DELETE`) et intégrité des données.  
- **Analyse de données immobilières** : extraction de statistiques sur les biens, les transactions et les clients.  
- **Documentation** : organisation des requêtes et des résultats de manière claire et compréhensible.

## Outils utilisés  
- **Système de gestion de base de données relationnelle (SGBDR)** : MySQL.  
- **SQL** : langage de requête pour créer, manipuler et interroger la base de données.  
- **Client SQL** : MySQL Workbench ou terminal MySQL pour exécuter les requêtes et visualiser les résultats.

## Résultats  
- Base de données immobilière créée avec plusieurs tables liées entre elles : biens, clients, transactions, agences.  
- Requêtes SQL réalisées pour obtenir :  
  - Les biens disponibles à la vente ou à la location.  
  - Les transactions par période et par type de bien.  
  - Les clients les plus actifs et leurs préférences.  
  - Les statistiques sur les zones géographiques et les types de biens.  
- Visualisation et synthèse des données pour des prises de décisions éclairées.

## Conclusion  
Ce projet m’a permis de maîtriser la conception et l’utilisation d’une base de données relationnelle dans un contexte concret. J’ai développé mes compétences en SQL et en modélisation des données, et j’ai appris à transformer des données brutes en informations structurées et exploitables pour l’analyse immobilière.
